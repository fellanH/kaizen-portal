M3: useIsMobile hydration mismatch fixed
Hook now returns undefined during SSR instead of coercing to false. Consumers handle the undefined state: sidebar provider uses nullish coalescing, layout treats undefined as falsy (mobile header hidden during SSR).
hooks/use-mobile.ts, components/ui/sidebar.tsx
M4: Project token exposed in URL query string fixed
Moved token from ?token=xxx query parameter to hash fragment (#xxx). Hash fragments are not sent in Referer headers and not stored in browser history in the same way. Dynamic route segments are incompatible with Next.js static export, so hash-based approach was chosen. Legacy query string still works as fallback.
projects/detail/project-detail.tsx, projects/page.tsx, messages/page.tsx
M5: Revision dialog allows empty submission fixed
Added trim() guard in handleRevision handler and disabled={!revisionMsg.trim()} on submit button.
projects/detail/project-detail.tsx
M6: Message send has no error handling fixed
Wrapped API call in try/catch. Text only cleared on success. toast.error shown on failure. MessageThread.handleSend also updated to preserve text on error.
projects/detail/project-detail.tsx
M7: Approve/Revision actions have no error handling fixed
Added catch blocks with toast.error() to both handleApprove and handleRevision. API failures now surface user-visible error messages.
